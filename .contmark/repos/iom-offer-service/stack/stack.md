## Runtime Stack

| Layer | Technology | Version / Notes |
|---|---|---|
| Language | Kotlin | 1.9.0 |
| JVM | Java | 17 |
| Framework | Spring Boot | 3.1.2 (WebFlux / reactive) |
| Web | Spring WebFlux | Reactive, non-blocking; `web-application-type: reactive` |
| Security | Spring Security OAuth2 | Dual-issuer JWT (Forgerock + Azure AD) |
| Persistence | Spring Data MongoDB Reactive | Database: `iom-offer`; collections: `offered_service_plans`, `search_request` |
| Messaging | Spring Kafka + Confluent Schema Registry | Consumer for `ServicePlanDomainEvent`; Avro deserialization |
| Resilience | Resilience4j | Circuit breaker per WebClient; retry via `Retry.backoff` |
| Observability | Micrometer + Zipkin | W3C trace propagation; Prometheus metrics |
| Feature Flags | LaunchDarkly | SDK key via `${launchdarklysdkkey}` |
| API Docs | SpringDoc OpenAPI | `/v3/api-docs`, `/webjars/swagger-ui` |
| Schema | Avro / Confluent Schema Registry | `ServicePlanDomainEvent.v3.avsc` generated by `generateAvroJava` |

## Build System

**Tool**: Gradle Kotlin DSL (`build.gradle.kts`)

### Key Gradle Tasks

| Task | Purpose |
|---|---|
| `test` | Run all unit tests |
| `compileKotlin` | Compile Kotlin sources |
| `jacocoTestReport` | Generate JaCoCo coverage report |
| `sonarqube` | Run SonarQube analysis |
| `bootRun` | Start service locally |
| `openApi` | Generate OpenAPI spec (source: build.gradle.kts) |
| `generateAvroJava` | Generate Java POJOs from Avro `.avsc` schemas (source: build.gradle.kts) |

## Environment Profiles

Spring profiles: `dev`, `sit`, `qa`, `uat`, `preprod`, `spt`, `prod`

On `sit`, `preprod`, `uat`: `OfferClientWithMockFallback` is active — falls back to mock offers service on primary API failure (source: src/main/kotlin/com/maersk/iom/offer/webclient/offer/v3/OfferClient.kt:OfferClientWithMockFallback)

## Auth Flow (Outbound)

All outbound WebClient calls use `client_credentials` OAuth2 grant via Spring Security auto-token-relay:
- `iomAuthProvider` / `offersAuthProvider` — Forgerock
- `ireAuthProvider` — routing client (hard-coded clientId in yml)
- `customsAuthProvider` — separate Forgerock client for customs API
